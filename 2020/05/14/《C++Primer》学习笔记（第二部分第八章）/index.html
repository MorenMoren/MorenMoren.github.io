  • 标准库使我们能忽略这些不同类型的流之间的差异，这是通过继承机制实现的。我们通常可以将一个派生类对象当作基类对象使用。

  • IO对象无拷贝或赋值：

  • 每个输出流都管理一个缓冲区，用来保存程序读写的数据。导致缓冲刷新（即数据真正写到输出设备或者文件）的原因有很多：

    1. 程序正常结束。
    2. 缓冲区满时，需要刷新缓冲，而后新的数据才能继续写入缓冲区。
    3. 可以使用操纵符如 endl 来显式刷新缓冲区。
    4. 在每个输出操作之后，我们可以用操纵符 unitbuf 设置流内部状态，来清空缓冲区。默认情况下，对 cerr 是设置 unitbuf 的，因此写到 cerr 的内容都是立即刷新的。
    5. 一个输出流可能被关联到另一个流。这种情况下，读写被关联的流时，关联到的流的缓冲区会被刷新。
