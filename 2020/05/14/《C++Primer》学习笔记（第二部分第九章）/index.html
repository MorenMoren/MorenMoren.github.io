​ 一个容器就是一些特定类型对象的集合。顺序容器为程序员提供了控制元素存储和访问顺序的能力。

一.顺序容器概述

  • 除了固定大小的array外，其它容器都提供高效，灵活的内存管理。我们可以添加和删除元素，扩张和收缩容器大小。

    1. string和vector随机访问是很快的，然而在中间位置添加或删除元素则很慢。
    2. list 和 forward_list 在任何位置添加或删除元素很快，然而随机访问很慢。
    3. deque支持快速的随机访问，同string和vector一样，它在中间添加或删除元素很慢。但是在两端添加或删除元素很快。

  • 以下是一些选择容器的基本原则：

    1. 除非有更好的理由，否则应使用vector。
    2. 如果程序有很多小元素，且空间额外开销很重要，则不要使用list或forward_list。
    3. 如果程序要求随机访问元素，应选择vector和deque。
    4. 如果要求中间插入删除元素，应使用list或者forward_list。
